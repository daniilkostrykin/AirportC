#  Система управления аэропортом

Консольное приложение на C# (.NET 10) для управления рейсами, пассажирами и продажей билетов. 
Проект разработан в учебных целях для демонстрации **трехслойной архитектуры**, принципов **ООП** и работы с **LINQ** и коллекциями. Основной акцент сделан на соблюдение принципа инверсии зависимостей (**D из SOLID**).

## Архитектура приложения (Три слоя)

Проект строго разделен на три изолированных слоя, взаимодействие между которыми происходит исключительно через интерфейсы:

1. **Слой представления (Console UI)**
   - Класс `AppMenu` реализует бесконечный цикл взаимодействия с пользователем.
   - **Не знает** о базе данных и репозиториях.
   - Зависит только от интерфейса бизнес-логики (`IAirportService`).

2. **Сервисный слой (Business Logic)**
   - Класс `AirportService` инкапсулирует бизнес-правила (например, проверку свободных мест при покупке билета).
   - Выполняет сложные выборки и формирование отчетов с помощью **LINQ**.
   - **Не создает** репозитории напрямую, а получает их через конструктор в виде интерфейсов (`IFlightRepository` и т.д.).

3. **Слой доступа к данным (Repositories)**
   - Выполняет роль базы данных в памяти.
   - Данные хранятся в строго типизированных коллекциях `List<T>`.
   - Выдает данные наружу в безопасном виде через `IReadOnlyList<T>`.

*Связывание слоев (Dependency Injection) происходит в точке входа приложения — файле `Program.cs`.*

##  Основной функционал

* **Управление рейсами:** добавление новых рейсов, просмотр списка рейсов по статусам (Ожидается, В полете, Завершен).
* **Управление пассажирами:** регистрация новых пассажиров.
* **Продажа билетов:** покупка билета на конкретный рейс с проверкой наличия свободных мест.
* **Отчетность (LINQ):**
    * Группировка и подсчет выручки по каждому рейсу (`GroupBy`, `Sum`, `Count`).
    * Вывод Топ-N самых прибыльных рейсов (`OrderByDescending`, `Take`).

## Применяемые технологии и концепции

* **Язык:** C# (.NET 10)
* **ООП:** Инкапсуляция (приватные поля, свойства), Композиция (билет связывает пассажира и рейс), Использование `enum` для статусов.
* **Коллекции:** Использование `List<T>` и `IReadOnlyList<T>`.
* **LINQ:** Активное использование операторов `Where`, `Select`, `GroupBy`, `Sum`, `Count`, `OrderByDescending`, `Take`.
* **Паттерны:** Repository, Dependency Injection (через конструктор), DTO (Data Transfer Object для возврата результатов статистики).
